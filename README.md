# EWUCSC - Authentication & Architecture



## 🔐 Authentication Flow Architecture

The authentication system uses a **hybrid security model** combining Firebase Authentication and custom JWT tokens.

### Flow Overview
1. **User Registers / Logs in**
2. **Firebase Authentication** validates credentials
3. **Firebase ID Token** is generated on the client
4. **Frontend** sends ID Token to the backend
5. **Backend** verifies token using **Firebase Admin SDK**
6. **MongoDB** user lookup or creation occurs
7. **Backend** generates a custom **JWT**
8. **JWT** is stored in `localStorage`
9. **Axios** attaches the JWT to all protected API requests

### Why This Architecture?

| Layer | Responsibility |
| :--- | :--- |
| **Firebase** | Identity verification & Secure password handling |
| **Backend** | Authorization, role control, and business logic |
| **MongoDB** | Persistent user data storage |
| **JWT** | Fast, stateless API authentication |

## 🔑 Tokens Used in the System

The application utilizes two distinct tokens to balance security and performance.

### 1️⃣ Firebase ID Token
Generated by Firebase immediately after login or registration.
*   **Used for:** `POST /api/users` and `POST /api/login`
*   **Purpose:** Verifies identity with the backend and prevents fake account creation.
*   **Header:** `Authorization: Bearer <firebase-id-token>`

### 2️⃣ Backend JWT Token
Generated by the server *after* successful Firebase verification.
*   **Stored in:** `localStorage` → `access-token`
*   **Used for:** Protected routes like `/api/profile`, `/api/homeworks`, and `/api/dashboard`.
*   **Header:** `Authorization: Bearer <jwt-token>`

---

## 🧠 Backend Security Middleware

### `verifyFirebase.js`
*   **Purpose:** Verifies the incoming Firebase ID Token.
*   **Usage:** During User Registration and Login.
*   **Result:** Attaches `req.firebaseUser` to the request.

### `verifyJWT.js`
*   **Purpose:** Verifies the custom backend JWT token.
*   **Usage:** Applied to all protected API routes.
*   **Result:** Attaches `req.user` to the request.
